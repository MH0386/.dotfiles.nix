# GEMINI Context: MH0386's NixOS Dotfiles

This project is a highly modular NixOS configuration repository using **Nix Flakes**, **Home Manager**, and **devenv**. It manages system and user-level configurations for multiple machines.

## Project Overview

- **Main Technologies:** Nix, NixOS, Home Manager, Nix Flakes.
- **Desktop Environment:** [niri](https://github.com/YaLTeR/niri) (a scrollable-tiling Wayland compositor) integrated with [DankMaterialShell](https://github.com/AvengeMedia/DankMaterialShell).
- **Target Hosts:** `MohamedDesktopNixOS`, `MohamedLaptopNixOS`.
- **Development Tooling:** [devenv](https://devenv.sh/) for consistent development shells and extensive pre-commit linting/formatting.

## Project Structure

- `flake.nix`: The entry point for the configuration. Defines inputs and `nixosConfigurations` for each host.
- `devenv.nix`: Configures the development environment, including `treefmt` and `git-hooks` (actionlint, nixfmt, deadnix, statix, ripsecrets, etc.).
- `nixos/`: Contains all NixOS-related configurations.
    - `configuration.nix`: Main entry point for common system-wide configuration.
    - `configuration/`: Modularized NixOS components (e.g., `boot.nix`, `networking.nix`, `users.nix`, `virtualisation.nix`).
    - `device/`: Host-specific configurations (e.g., `hardware-configuration.nix`, per-machine overrides).
    - `home-manager/`: User-level configurations using Home Manager.
        - `hm-configuration.nix`: Main entry point for user `mohamed`.
        - `hm-configuration/`: Modularized Home Manager components (e.g., `gtk.nix`, `programs.nix`, `ghostty/`, `niri/`).
        - `dotfiles/`: Raw configuration files managed by Home Manager (e.g., `niri/config.kdl`).

## Key Commands

### Building and Applying Configuration

To apply the configuration for the current host (assuming the hostname matches one of the defined `nixosConfigurations`):

```bash
sudo nixos-rebuild switch --flake .
```

To apply for a specific host (e.g., Desktop):

```bash
sudo nixos-rebuild switch --flake .#MohamedDesktopNixOS
```

The project also suggests using `nh` (Nix Helper) for cleaner management:

```bash
nh os switch .
```

### Development Environment

Enter the development shell:

```bash
devenv shell
```

Format the entire codebase:

```bash
treefmt
```

## Architecture & Conventions

- **Modularity:** Configuration is split into small, logical files located in `nixos/configuration/` and `nixos/home-manager/hm-configuration/`.
- **Special Args:** The flake passes `inputs`, `pkgsStable`, `nur`, and other specialized flakes (like `niri`, `dms`, `nix-flatpak`) as `specialArgs` and `extraSpecialArgs` to both NixOS and Home Manager modules.
- **Nix Configuration:**
    - **Experimental Features:** `flakes` and `nix-command` are enabled.
    - **Binary Caches:** Uses multiple substituters including `cuda-maintainers`, `nix-community`, `devenv`, `flox`, and `garnix` for faster builds.
    - **Unfree Software:** Enabled globally with `allowUnfree = true`.
- **System Features:**
    - **CUDA:** `cudaSupport` is enabled globally.
    - **State Version:** `25.11` (bleeding edge).
    - **Auto-upgrades:** Enabled and scheduled daily with auto-reboot.
    - **Optimization:** `zramSwap` is enabled.
    - **Fonts:** Primarily uses `monaspace`.

## Hardware Config
 Each device in `nixos/device/` should have its own `hardware-configuration.nix` (usually generated by `nixos-generate-config`).
